## Question 1
**Q:** How would you define Machine Learning?

**A:** Machine Learning is a system that learns patterns or rules from data or
past experience, rather than being explicitly programmed with rules by a
human, and whose performance at a task can be measured and shown to improve
with that experience.

## Question 2
**Q:** Can you name four types of problems where it shines?

**A:** Machine learning shines in four types of problems:
- **Fluctuating environments** — fraud detection, where tactics keep changing, so a system needs to adapt rather than rely on fixed rules.
- **Long lists of hand-tuned rules** — fraud detection again: if/else rules would balloon in complexity over time, so ML learns the patterns instead of us hardcoding them.
- **Complex problems with no algorithmic solution** — warehouse inventory management, where the relationships between demand, seasonality, and supply are too tangled for a human to express as explicit logic.
- **Helping humans learn (data mining)** — surfacing hidden patterns in large datasets that a person couldn't find by eyeballing the data themselves.

## Question 3
**Q:** What is a labeled training set?

**A:** A labeled training set is a dataset that contains both features and labels for each example — the features are the input variables, and the label is the correct answer/output you're trying to predict. An unlabeled dataset, by contrast, contains only features and no labels.

## Question 4
**Q:** What are the two most common supervised tasks?

**A:** The two most common supervised tasks are:
1. **Classification** — predicting a category/label (e.g., spam or not spam).
2. **Regression** — predicting a quantity/number (e.g., stock price, how many units to reorder).

## Question 5
**Q:** Can you name four common unsupervised tasks?

**A:** The four common unsupervised tasks are:
1. **Clustering** — grouping similar instances together by shared features (e.g., grouping similar products so they're easier to find), with no predefined labels.
2. **Anomaly detection** — spotting instances that don't fit the normal pattern (e.g., flagging transactions that look unusual compared to typical spending behavior).
3. **Dimensionality reduction** — compressing many features into fewer while preserving most of the useful information (e.g., reducing 200 transaction features down to 10 without losing much signal).
4. **Association rule learning** — discovering relationships between attributes that weren't explicitly programmed (e.g., noticing that customers who buy diapers also tend to buy beer).
